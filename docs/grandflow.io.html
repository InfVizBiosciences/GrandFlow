class grandflow.io.FastaIO.FastaReader(f)[source]¶

Bases: grandflow.io.base.ReaderBase

Streaming reader for FASTA files, useable as a one-shot iterator over FastaRecord objects. Agnostic about line wrapping.

Example:

>>> from grandflow.io import FastaReader
>>> from grandflow import data
>>> filename = data.getTinyFasta()
>>> r = FastaReader(filename)
>>> for record in r:
...     print record.header, len(record.sequence)
ref000001|EGFR_Exon_2 183
ref000002|EGFR_Exon_3 203
ref000003|EGFR_Exon_4 215
ref000004|EGFR_Exon_5 157
>>> r.close()

Random-access FASTA file reader.

class grandflow.io.FastqIO.FastqRecord(header, sequence, quality=None, qualityString=None)[source]¶

Bases: object

A FastqRecord object models a named sequence and its quality values in a FASTQ file. For reference consult Wikipedia's FASTQ entry. We adopt the Sanger encoding convention, allowing the encoding of QV values in [0, 93] using ASCII 33 to 126. We only support FASTQ files in the four-line convention (unwrapped). Wrapped FASTQ files are generally considered a bad idea as the @, + delimiters can also appear in the quality string, thus parsing cannot be done safely.

If f is a path to a true FOFN on disk, any paths listed in the FOFN that are relative (i.e., do not contain a leading '/') will be reckoned from the directory containing the FOFN.

Class for GFF record, providing uniform access to standard GFF fields and attributes.

>>> from grandflow.io import Gff3Record
>>> record = Gff3Record("chr1", 10, 11, "insertion",
...                     attributes=[("foo", "1"), ("bar", "2")])
>>> record.start
10
>>> record.foo
'1'
>>> record.baz = 3
>>> del record.baz

Attribute access using record.fieldName notation raises ValueError if an attribute named fieldName doesn't exist. Use:

>>> record.get(fieldName)

to fetch a field or attribute with None default or:

>>> record.get(fieldName, defaultValue)

to fetch the field or attribute with a custom default.

grandflow.io.rangeQueries.getOverlappingRanges(tStart, tEnd, nBack, nOverlap, rangeStart, rangeEnd)[source]¶

Return indices overlapping the range defined by [rangeStart, rangeEnd). Here tStart, tEnd, nBack, nOverlap are vectors of length n sorted according to tStart and tEnd. The vectors nBack and nOverlap are typically produced by computeIndices[DP].

grandflow.io.rangeQueries.getReadsInRange(cmpH5, coords, justIndices=False)[source]¶

Return an ndarray representing the portion of the reads which overlap the range specfied by coords, where coords is a three-tuple composed of (refSeqID, rangeStart, rangeEnd). Here, cmpH5 is an hdf5 object representing a pointer to a sorted cmp.h5 file.

grandflow.io.rangeQueries.leftmostBinSearch(vec, val)[source]¶

Return the leftmost position in the vector vec of val. If val is absent then we return the lefternmost position for the value: max(vec[vec < val]). The time complexity here is potentially worse than log(n) because of the extra step of walking backwards.

grandflow.io.rangeQueries.makeReadLocator(cmpH5, refSeq)[source]¶

grandflow.io.rangeQueries.projectIntoRange(tStart, tEnd, winStart, winEnd)[source]¶

Find coverage in the range [winStart, winEnd) implied by tStart, tEnd vectors. Coverage can be most efficiently calculated by first obtaining all reads overlapping the range using the getOverlappingRanges function then projecting them into the same or smaller range

grandflow.io.rangeQueries.rightmostBinSearch(vec, val)[source]¶

Return the rightmost position in the vector vec of val. If val is absent then we return the leftmost position of the value: min(vec[vec > val]). If val is greater than all elements in vec we return len(vec).
